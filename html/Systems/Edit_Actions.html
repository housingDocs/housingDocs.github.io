Edit Actions
Power all your imaginations with Interactives
Introduction
Edit Actions allow Housing owners to customize interactions with clickable items, NPCs, and regions.
Using this feature, you can set commands, messages, or triggers when players interact with specific objects—enhancing gameplay, roleplay, or minigame mechanics.
Key Features:
  • Item Actions – Run commands or send messages when players click items.
  • NPC Interactions – Customize dialogue, teleports, or permissions when talking to NPCs.
  • Region Triggers – Execute actions when players enter/exit designated areas.
This system provides deeper control over your Housing experience, making builds more dynamic and interactive.
Fundamentals
When will we use Edit Actions?
Edit Actions is used everywhere around Housing Coding: Region Execution, Event Actions, Commands, Functions, Click Action in Custom Menus, Action Buttons // Pads // NPCs.
Remove Last Removed Action
Marked as a cauldron in an action editor, it restores the action you last removed.
How to add // edit // remove actions?
To add an action, head into the action editor, and click Add Action (Marked as paper).
To edit an action, left click on the selected action. Press and hold SHIFT, then left/right click to change its order.
To remove an action, right click on the selected action.
Actions Explained
Edit Actions currently contain 36 main functions (excluding Conditionals). Below is each component explained in detail:
1. Change Player's Group
Marked with: Player Head
Changes the group of a target player.
It contains 2 settings: Target Group and Demotion Protection.
Group: When the action is executed, the target player's group is set to the set target group.
Demotion Protection: When enabled, the target player's group WILL NOT be changed when the target group has a lower group priority than that of the player's original group.
To learn more about Group Priorities, head to Permissions and Groups page.
2. Kill Player
Marked with: Iron Bars
Kills the target player.
This action will NOT work when the target player is in Creative Mode!!!
3. Full Heal
Marked with: Golden Apple
Heals the target player back to full health.
4. Display Title
Marked with: Book
Displays a title to the target player on the middle of their screen.
It contains 5 settings: Title, Subtitle, Fadein, Stay, and Fadeout
Title: Displayed as the primary text on the screen.
Subtitle: Displayed as the secondary text on the screen, below Title.
Fadein: Counted in seconds, determines how much time it takes for the title to gradually appear.
Stay: Counted in seconds, determines how long the title should stay on screen.
Fadeout: Counted in seconds, determines how much time it takes for the title to gradually disappear.
5. Display Action Bar
Marked with: Book and Quill
Displays a message to the target player at a position above the player's hotbar.
6. Reset Inventory
Marked with: Stone
Resets a player's inventory.
If a default inventory layout is set, the player's inventory will be reset to the default inventory layout.
7. Change Max Health
Marked with: Dandelion
TChanges a player's maximum health.
It contains 3 settings: Max Health, Mode, and Heal On Change.
Max Health: Specifies a set variable. (Default: 20)
Mode: Specifies which algorithmic action to perform on the variable. (Set, Increment, Decrement, Multiply, Divide)
Heal on Change: When the maximum health of the player is changed, heal the target player to full health.
8. Parkour Checkpoint
Marked with: Lightweighted Pressure Plate (Gold)
If a parkour is present in the Housing, teleports the player to the last parkour checkpoint they reached.
If the target player hasn't started a parkour, teleports the player to the parkour start plate.
9. Give Item
Marked with: Chest
Gives the target player a selected item.
It contains 4 settings: Item, Allow Multiple, Inventory Slot, and Replace Existing Item.
Item: Specifies the selected item given to the player. Stacking items in the variable will give the player that amount.
Allow Multiple: When disabled, if the target player already has the target item, it won't be given again.
Inventory Slot: Specifies where the item appears in the target player's inventory. It can be any inventory slot.
Replace Existing Item: When enabled, if an item currently occupies the inventory slot it will be replaced with the new item.
10. Remove Item
Marked with: Cauldron
Removes a selected item from the targeted player. Stacking items in the variable will remove that amount from the player.
If the player has insufficient amount of items, it will be fully cleared.
11. Send a Chat Message
Marked with: Empty Map
Sends a message in chat to the target player.
12. Apply Potion Effect
Marked with: Water Bottle
Applies a potion effect to the targeted player.
It contains 5 settings: Effect, Duration, Level, Override Existing Effects, and Show Potion Icon.
Effect: Determines the potion effect the target player would get.
Duration: Counted in seconds, determines the longevity of the potion effect. (Maximum: 2592000)
Level: Specifies ntensity of the effect. (Maximum: 10)
Override Existing Effects: When enabled, replaces the current potion effect if the target player already has the selected potion effect.
Show Potion Icon: When enabled, shows the potion icon on the top right of the Minecraft Client for players running version 1.9 or later.
13. Clear All Potion Effects
Marked with: Empty Bottle
Clears all potion effects of the targeted player.
14. Give Experience Levels
Marked with: Experience Bottles
Adds XP levels to the target player.
15. Send to Lobby
Marked with: Bookshelf
Sends a player to any selected lobby in Hypixel.
Due to Owner Protection, Housing owners will not be affected by this action.
16. Change Variable
Marked with: Feather
Changes a variable stat of the targeted player. It can be a number or a string.
It contains 5 settings: Holder, Variable, Mode, Value, and Automatic Unset.
Holder: Specifies which type of variable will be changed. (Player, Global, Team)
Variable: Specifies the name of the target variable.
Mode: Specifies which algorithmic action to perform on the variable. (Unset, Set, Increment, Decrement, Multiply, Divide)
Value: Specifies the value to be assigned to the variable.
Automatic Unset: When enabled, values that are considered default will unset the variable automatically.
17. Teleport Player
Marked with: Ender Pearl
Teleports the target player to a set location.
It contains 2 settings: Location and Prevent Teleporting Inside Blocks.
Location: Specifies the location the target player gets teleported to.
Options include: House Spawn Location, Location of the player activating the function, Location of the coder of the function, and Custom Coordinates.
Prevent Teleporting Inside Blocks: If the target coordinate is within a block, the target player automatically gets teleported upwards until there's sufficient space.
18. Fail Parkour
Marked with: Lightweighted Pressure Plate (Gold)
Ends the target player's timed parkour session. (IF applicable)
A reason for ending the session can also be provided to the target player.
19. Play Sound
Marked with: Note Block
Gives the target player a selected item.
It contains 4 settings: Sound, Volume, Pitch, and Location.
Sound: Specifies the sound to be played. Must be a vanilla Minecraft sound.
Volume: (Range: 0-2), determines the volume of the sound.
Pitch: (Range 0-2), determines the pitch of the sound.
Location: Specifies where would the sound be played.
20. Set Compass Target
Marked with: Compass
Specifies where would the compass of a target player point towards. Input must be a coordinate.
21. Set Gamemode
Marked with: Daylight Detector
Specifies what gamemode will the target player obtain. (Adventure // Survival // Creative)
22. Change Health
Marked with: Red Apple
Specifies the amount of health the target player gets.
It contains 2 settings: Health and Mode.
Health: Specifies a specific variable. (Default: 20).
Mode: Specifies which algorithmic action to perform on the variable. (Set, Increment, Decrement, Multiply, Divide)
23. Change Hunger Level
Marked with: Cooked Steak
Specifies the amount of hunger/saturation level the target player gets.
It contains 2 settings: Level and Mode.
Level: Specifies a specific variable. (Default: 20).
Mode: Specifies which algorithmic action to perform on the variable. (Set, Increment, Decrement, Multiply, Divide)
24. Random Action
Marked with: Ender Chest
Executes a single random action from a list of selected actions.
Every Edit Action is allowed to be randomized within except Conditionals.
25. Trigger Function
Marked with: Activator Rail
Triggers a pre-defined function. (/functions)
26. Apply Inventory Layout
Marked with: Iron Axe
Applies a pre-defined inventory layout to the target player.
27. Enchant Held Item
Marked with: Enchanted Book
Enchants the item held by the target player.
It contains 2 settings: Enchantment and Level.
Level: Specifies a target enchantment. All enchantments from Minecraft 1.8 are included.
Mode: Specifies the intensity of the enchantment. (Range: 1-10)
28. Pause Execution
Marked with: Red Bed
Pauses execution for the remaining actions for a specific amount of time. (Counted in ticks, 20 ticks = 1 second)
29. Set Player Team
Marked with: Wooden Sign
Allocates the target player to a team.
30. Display Menu
Marked with: Chest
Displays a custom menu to the target player.
31. Drop Item
Marked with: Dropper
Drops an item at a specific location.
It contains 6 settings: Item, Location, Drop Naturally, Disable Item Merging, Prioritize Player, and Fallback to Inventory.
Item: Specifies a target item / block.
Location: Specifies a target location for the item to appear.
Drop Naturally: When enabled, the item will be dropped naturally in the Housing, having slight randomness to its location.
Disable Item Merging: When enabled, the item will not merge with other items on the ground.
Prioritize Player: When enabled, the player the action is executed on will be prioritized for picking up the item.
Fallback to Inventory: When enabled, items will be put into the player inventory if they can't be dropped in the Housing.
32. Change Velocity
Marked with: Slime Block
Changes the velocity direction of the player.
It contains 3 settings: X Velocity, Y Velocity, and Z Velocity.
Each direction and velocity value should be determined using the coordinates system (F3)
33. Launch to Target
Marked with: Slime Ball
Launches the player in the general direction of the provided location.
It ontains 2 settings: Target Location and Launch Strength.
Target Location: Specifies the target location.
Demotion Protection: Specifies the strength of launching the target player.
34. Set Player Weather
Marked with: Water Bucket
Changes the weather for a target player. This will result in the weather being different from the world while set. (Sunny // Raining)
35. Set Player Time
Marked with: Clock
Changes the time of the world for a target player.
36. Toggle Nametag Display
Marked with: Water Bucket
Toggles whether the player's nametag is shown or hidden to other players in the Housing. (On // Off)
Conditionals
Marked with: Redstone Dust
(IF-THEN-ELSE) Use conditions to only execute provided actions when conditions are met by the player.
It contains 4 settings: Conditions, Match Any Condition, If Actions, and Else Actions.
Conditions: Specifies the selected item given to the player. Stacking items in the variable will give the player that amount.
Match Any Condition: When enabled, only one condition needs to match for IF ACTIONS to be performed.
If Actions: Actions that will run when the conditions are met.
Else Actions: Actions that will run if the conditions are not met.

Each condition has an invert option, which changes the condition to activate when the condition is not met.
