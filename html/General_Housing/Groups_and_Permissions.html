Permissions and Groups
Control player mechanics using Permissions and Groups
Introduction
Introduced along with Housing, Permissions and Groups is a critical feature used to manage Housings.
Permissions determine what actions a player can perform in a Housing plot, such as building, using items, or interacting with certain mechanics.
Groups are categories that players are assigned to, each with different permission levels.

Why are Permissions important?
  • Security: Prevents griefing (unauthorized building/destruction).
  • Organization: Helps manage different types of visitors (e.g., guests, co-owners).
  • Customization: Allows for roleplay setups (e.g., staff ranks, VIPs).
  • Automation: Some permissions allow players to bypass certain restrictions (e.g., teleporting, using commands).
Fundamentals
Permissions and Groups is split into multiple functions:
Group Limit
Each Housing has a limit of 20 groups. As of now, it is not possible to extend the limit.
Group Name
Group Name is visible while editing groups, during Action Editing, or when changing groups of other players. It is not visible in normal gameplay.
The maximum length of a group name is 16 characters.
Group Tag
Group Tag is visible in normal gameplay, and is shown as a suffix (after the player's username) on player's nametags.
The maximum length of a group name is 10 characters.
It is also an option to toggle showing the Group Tag in chat.
Group Tags are controlled by Group Color, which is explained below.
Group Color
Group Color pairs along with Group Name and Group Tag. It assigns a color to each group.
There are 16 colors to choose from: Black, Dark Blue, Green, Dark Cyan, Dark Red, Dark Purple, Gold, Gray, Dark Gray, Blue, Lime, Cyan, Red, Light Purple, Yellow, and White.
Different groups can have the same Group Color.
Group Priority
Group Priority, in simpler terms, decides the ranking of each group in the Housing.
Group Priority can be assigned as a number ranged 1 to 20.
More important roles (e.g., admins) should be assigned a higher priority.
This feature is crucial for a lot of usecases, such as assigning priority for Commands and Conditionals.
Set Default Group
Marked as a diamond in the group editing page, it sets a selected group as default.
When players join the Housing for the first time, or joins without a non-default group, they will be assigned with the new default group.
BE CAREFUL! Deleting a group will reset the default group to the group with the lowest priority.
Clear Group Players
Marked as a cauldron in the group editing page, it clears all players from the selected group, including offline players.

Managing Groups
How to Create/Edit/Delete a Group?
To create a group, navigate to: Housing Menu → Permissions and Groups → Create Group (Marked as an Oak Sign)
To edit a group, navigate to: Housing Menu → Permissions and Groups → A group you have currently (Marked as a color-coded player head)
To delete a group, navigate to: Housing Menu → Permissions and Groups → A group you have currently → Delete Group (Marked as a TNT block)

BE CAREFUL! Deleting a group will reset the default group to the group with the lowest priority.
